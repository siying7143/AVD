GET/api/vulnerabilities

Returns a paginated list of published vulnerability records using the same filters available in the database page.

GET/api/vulnerabilities/{cve_id}

Returns one normalised CVE record with score breakdown, evidence fields, taxonomy lists and external references.

Supported query parameters

qGlobal search across CVE, description, vendors, products and CWE.
cveCVE ID partial match.
vendorVendor JSON/text field partial match.
productProduct JSON/text field partial match.
priorityRepeatable AVD priority values such as critical, high, medium or low.
severityRepeatable NVD severity values.
score_min / score_maxAVD final score range, clamped to 0–10.
base_score_min / base_score_maxNVD CVSS base score range.
kevall, yes or no.
au_relatedall, yes or no.
page / page_sizePagination controls. Page size supports 5–100.

Example requests

/api/vulnerabilities?priority=critical&sort=score_desc
/api/vulnerabilities?kev=yes&au_related=yes
/api/vulnerabilities?q=remote%20code%20execution&page_size=10
/api/vulnerabilities/CVE-2025-0001
