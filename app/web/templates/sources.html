Source pipeline

Data sources

AVD uses the existing project pipeline: NVD provides the core CVE records; CISA KEV, EPSS and Australian cybersecurity source matches provide prioritisation evidence.

Methodology
NVD

Core CVE records

Descriptions, CVSS base scores, severity, CWE identifiers, vendors, products, publication dates and modification dates.

KEV

Known exploited evidence

CISA Known Exploited Vulnerabilities matches are used as high-confidence exploitation evidence in the AVD score.

EPSS

Exploitation probability

EPSS score and percentile are displayed where available and used as exploitation-likelihood evidence by the enrichment pipeline.

AU

Australian contextual signal

ACSC / ASD / cyber.gov.au related matches are treated as local-context signals. They highlight triage relevance, not guaranteed Australia-specific exploitation.

Implemented workflow

How records move through the database

1Import NVD records
2Create or update AVD entries
3Compute AVD assessment
4Attach KEV / EPSS / AU evidence
5Publish to the web portal and API
